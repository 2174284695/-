# 路线模拟器 v1.4.0

本版本修复道路吸附点击无响应，并新增真正的多途经点路线规划。

## 修复

- 修复路线不足两个点时道路按钮被禁用、点击没有任何反馈的问题
- 道路按钮现在始终可点击，条件不足时会明确提示下一步操作
- 请求开始后立即显示“正在请求道路路线”和“规划中”状态
- 请求失败时显示具体错误并保留原路线
- 点选途经点与路由服务返回的道路几何分开存储，避免重新规划时混用

## 多点路线规划

1. 选择“点选”模式。
2. 按希望经过的先后顺序，在地图上添加 2～25 个蓝色途经点。
3. 点击“多点规划”。
4. 选择“步行路线”或“驾车路线”。
5. 应用会生成依次经过全部所选点的沿道路路线。

途经点会单独保存在设备本地。规划完成后仍可切换出行方式再次规划；点击“撤销”可以
恢复规划前路线。

## 手绘道路吸附

选择“手绘”画出大致路线后，点击“手绘吸附”，选择步行或驾车道路。应用会从手绘
轨迹中优先抽取转弯和形状变化明显的代表点，并生成尽量贴合整条手绘意图的道路路线。
该功能处理的是连续手绘轨迹，并不要求用户逐个添加途经点。

## 道路服务

道路规划使用 OpenStreetMap.de 提供的 OSRM 公共服务。只有主动执行道路功能时，最多
25 个路线坐标才会发送给该服务。公共服务可能限流或暂时不可用，正式或高频使用建议
自建 OSRM 服务。

## 兼容性边界

应用继续使用 Android 官方测试位置提供器，不隐藏模拟位置标记，也不绕过第三方应用
的安全检测。目标应用可以自行识别并拒绝模拟位置，因此无法保证所有第三方软件接受
模拟坐标。
