# 路线模拟器 v1.5.0

本版本针对“道路吸附点击后没有结果”重新设计了网络和交互流程，并修复底部菜单遮挡
定位按钮的问题。

## 道路吸附修复

- 复杂手绘路线不再一次性请求，改为每批最多 8 个代表点的分段规划
- 相邻分段共享连接点，返回结果会自动合并为连续路线
- 主 OSRM 服务失败时，自动切换到独立的 Valhalla 路由引擎
- 步行和驾车模式都支持主服务与备用服务切换
- 新增 Valhalla polyline6 路线解码及对应单元测试
- 道路请求开始后持续显示进度对话框，不会再像“点击没反应”
- 两套服务都失败时弹出完整诊断信息，包含主服务和备用服务的错误
- 失败时不会修改或清空原路线

## 布局修复

- 缩放和定位控件不再使用固定底部偏移
- 根据底部菜单实际高度动态上移
- 支持不同屏幕高度和系统字体缩放，避免定位按钮被菜单遮挡

## 实际服务验证

已使用上海坐标验证：

- OpenStreetMap.de OSRM 步行路线返回成功
- OpenStreetMap.de OSRM 驾车路线返回成功
- OpenStreetMap.de Valhalla 步行备用路线返回成功

手机仍需能够访问这些在线服务。若两套服务均被当前网络拦截，应用会直接显示具体错误。

## 使用

### 连续手绘吸附

1. 选择“手绘”。
2. 用手指连续画出完整的大致轨迹。
3. 点击“手绘吸附”。
4. 选择步行或驾车道路。
5. 等待进度对话框关闭并显示结果。

### 多点路线规划

1. 选择“点选”。
2. 按顺序添加 2～25 个蓝色途经点。
3. 点击“多点规划”。
4. 选择步行或驾车路线。
