# 路线模拟器

用于 Android 开发、导航原型和定位场景测试的开源路线模拟应用。

应用通过 Android 官方的“选择模拟位置信息应用”机制工作，不需要 root，也不包含
隐藏模拟标记、绕过第三方应用检测或规避 Android 安全机制的功能。

## 功能

- 使用 MapLibre 浏览地图，支持手势和按钮缩放至 20 级
- 在地图上点选路线，或用手指连续绘制路线
- 点选多个途经点后，可拖拽蓝色或橙色途经点调整通过顺序，长按可切换类型或删除
- 途经点显示顺序编号，支持弹窗精确上移/下移、一键反转起终点顺序
- 多点规划会自动区分道路点和强制经过点，远离道路的点不会被规划服务吞掉
- 强制经过点相邻路线段会以橙色虚线显示，并在模拟时自动降速
- 支持一键将手机真实当前位置设为路线或途经点起点
- 自动抽稀手绘轨迹，减少无意义的密集坐标点
- 已有路线可继续手绘完善，支持从终点继续画，也支持从起点前补画
- 按每次点选或每一笔手绘撤销
- 将手绘路线吸附为步行道路
- 选择多个途经点，按点选顺序规划完整步行路线，并保留必要的非道路经过点
- 也可直接把当前屏幕路线作为输入重新规划为步行道路路线
- 支持一键勾选往返路线，到达终点后按原路线返回起点
- 底部面板按路线状态组织，显示当前状态和下一步建议
- 主界面保留编辑方式切换、路线模式、速度和主动作
- 更多工具按“绘制与规划 / 路线管理 / 模拟高级”分组展开
- 底部操作面板可收起，减少对地图查看的遮挡
- 底部操作面板会限制最大高度，功能较多时可在面板内滚动
- 一键定位到手机当前真实位置
- 命名保存路线，并在路线库中载入或删除
- 自定义目标速度（1.0～30.0 km/h）
- 在“模拟高级”中自定义自然速度波动（±1%～±15%）
- 使用平滑随机漂移和加速度限制模拟自然移动
- 默认模拟起步加速、到达减速、转弯减速和长直线略加速
- 根据实时移动距离和动态步幅累计模拟步数
- Android 14+ 可在用户授权后将步数同步到 Health Connect
- 使用前台服务持续模拟，可从通知栏停止
- 自动保存当前路线、路线库、速度和波动设置
- 应用回到前台时自动恢复地图，必要时重新加载地图样式

## 系统要求

- Android 8.0（API 26）或更高版本
- 已开启系统开发者选项
- 可访问互联网，以加载在线地图
- Android Studio 自带的 JDK 17 或其他兼容 JDK

Health Connect 步数同步仅支持 Android 14（API 34）或更高版本，并且需要用户明确授权。

## 使用方法

1. 使用 Android Studio 打开项目并安装 `app`。
2. 在手机中开启“开发者选项”。
3. 进入“选择模拟位置信息应用”，选择“路线模拟器”。
4. 开启系统定位服务，并授予应用精确位置权限。
5. 点击地图右侧“定位”，可直接移动到手机当前真实位置。
6. 查看底部状态提示条，根据“当前状态 + 下一步建议”选择操作。
7. 在“编辑方式”中选择“定位设起点”“点选”或“手绘”，在地图上创建路线。
8. 手绘路线可在“更多工具 > 绘制与规划”中点击“道路吸附”，转换为步行道路。
9. 点选模式下，可拖动蓝色或橙色途经点调整顺序，长按某个途经点可改为道路点、强制经过点或删除。
10. 也可以在“更多工具 > 绘制与规划”中点击“顺序编辑”或“反转顺序”。
11. 多点路线可按点选顺序添加途经点，再在“主动作”中点击“多点规划”。远离道路的途经点会变为橙色强制经过点。
12. 已载入或当前显示的路线，也可以直接点击“多点规划”重新生成步行道路路线。
13. 需要到达终点后按原路返回起点时，勾选“路线模式：往返”。
14. “路线库”在主动作区；“保存路线 / 撤销 / 清空 / 显示全程”在“更多工具 > 路线管理”。
15. 设置目标速度；自然速度波动和 Health Connect 位于“更多工具 > 模拟高级”。
16. 点击“开始模拟”。

停止模拟后，应用会移除测试位置提供器，让系统恢复真实定位。

如果从 1.0 版本升级后，应用仍未出现在“选择模拟位置信息应用”列表中，请卸载旧版
后重新安装。1.1 及更高版本已经声明 Android 所需的模拟位置权限。

## 模拟步数

应用根据路线的实际推进距离和当前速度估算动态步幅，并在界面中实时累计模拟步数。
在 Android 14 或更高版本上，可以开启“同步模拟步数到 Health Connect”，授权后应用
会以自身为数据来源写入短时间段步数记录。

普通 Android 应用不能向系统硬件 `TYPE_STEP_COUNTER` 传感器注入事件。因此，不支持
Health Connect 或只读取硬件计步传感器的第三方应用，不一定会显示这些模拟步数。

服务。

## 道路吸附与路线规划

应用提供两种道路功能：

- **道路吸附**：先用手指连续画出完整的大致轨迹，应用会分析整条手绘线，优先保留
  转弯和形状变化明显的位置，再自动吸附为步行道路，并非只处理点选路线。
- **手绘补线**：已有路线时，可以从当前终点继续手绘，也可以在当前起点前补画一段；
  应用会自动识别靠近路线端点的手绘线并接到正确一端。
- **多点规划**：选择“点选”，依次添加 2～25 个途经点，再按点选顺序生成步行路线。
  默认新点先作为自动道路点处理；规划完成后，若某个自动点距离道路路线约 10 米以上，
  会变为橙色强制经过点，并把原始坐标插回最终路线，避免校园操场、广场、楼前区域等
  非道路点被直接忽略。若当前没有途经点，但屏幕上已有路线，会直接使用当前路线作为
  规划输入。
- **途经点编辑**：蓝色表示道路点，橙色表示强制经过点。途经点会显示顺序编号。在
  “点选”模式下，拖动途经点可以调整通过顺序，长按途经点可以手动设为道路点、强制
  经过点或删除该点；也可以通过“顺序编辑”弹窗精确上移或下移。
- **离路线段提示**：强制经过点相邻的路线段会用橙色虚线覆盖显示。模拟经过这些段时，
  速度会自动降低，并在进入/离开前后做短距离过渡，避免非道路穿越段仍按道路速度移动。
- **定位设起点**：把手机当前真实位置插入为路线起点。已有途经点时会成为第 1 个
  途经点；只有普通路线时会插到路线最前方。
- **反转顺序**：一键反转当前路线方向，对调起点和终点；规划后的道路几何也会整体
  反向，已有途经点时只同步反转编号顺序。
- **往返路线**：勾选后不会改写原始路线，只在模拟时临时追加返程，到达终点后按原路返回起点。

## 主界面结构

底部面板按路线状态组织：

- **状态提示条**：显示当前是空路线、已有路线、混合路线、规划中还是模拟中，并给出下一步建议。
- **编辑方式**：提供定位设起点、点选、手绘和浏览，空路线时可从这里开始创建路线。
- **路线模式**：往返路线作为清晰开关保留在主界面。
- **主动作**：开始模拟、多点规划、路线库和更多工具。
- **更多工具**：展开后分为“绘制与规划”“路线管理”“模拟高级”，自然速度波动和 Health Connect 位于模拟高级。

## 速度模拟

速度模拟默认包含多层自然化处理：

- 在用户设定速度附近进行平滑随机波动。
- 起步阶段逐渐加速，接近终点时逐渐减速。
- 检测到路线前后方向变化较大时自动降低速度。
- 长直线路段会略微加速，但仍受整体速度波动和加速度限制约束。
- 经过橙色强制经过点相邻的离路线段时自动降速。

执行规划时，应用会：

1. 从当前路线中按顺序选取最多 25 个代表点。
2. 请求 OpenStreetMap.de 提供的 OSRM 步行路由服务。
3. 使用服务返回的完整道路几何替换当前路线。
4. 把吸附前路线保留在撤销历史中，点击“撤销”即可恢复。

按钮始终可以点击。路线点不足时会说明需要的操作；请求开始后按钮会显示“规划中”，
并显示不可忽略的进度对话框。复杂手绘路线会分段请求，主 OSRM 服务失败时自动切换
到独立的 Valhalla 备用引擎。两套服务都失败时会保留原路线，用红线标出本次失败的
大概规划范围，并提供“重新规划 / 保留原路线 / 撤销规划”三个选择，不会静默无响应。

地图右侧的缩放和定位控件会根据底部菜单的实际高度自动上移。底部菜单会限制最大高度，
内容较多时可在面板内滚动；也可以点击“收起”折叠，只保留一行标题和展开按钮，方便
查看地图。

道路吸附需要联网。公共路由服务可能限流、暂时不可用，或因当地道路数据不完整而无法
生成理想路线。大量使用或正式部署时，应改为自建 OSRM 服务或接入具有明确服务保障的
路由供应商。默认端点定义在 `RoadRouteClient.java`。

## 数据与隐私

路线、路线库和设置默认保存在设备本地，不会发送给项目维护者。

当前编辑中的途经点类型会随应用状态保存；新保存到路线库的命名路线也会保存道路点/
强制经过点元数据，载入后会恢复蓝色/橙色途经点。旧版本已经保存的路线没有这部分
元数据，载入后仍会保留路线几何，再次规划时会按坐标重新自动判断途经点类型。

- 地图服务会收到加载地图切片所需的常规网络请求。
- 只有用户主动点击“道路吸附”或“多点规划”时，最多 25 个代表性路线坐标才会发送到
  `routing.openstreetmap.de`，用于计算道路路线。
- 开启 Health Connect 同步后，估算步数会按用户授权写入设备上的 Health Connect。

## 权限说明

| 权限 | 用途 |
| --- | --- |
| 精确位置 | 一键定位真实位置，以及运行 Android 定位前台服务 |
| 模拟位置 | 被 Android 开发者选项选为模拟位置应用 |
| 网络和网络状态 | 加载在线地图，并在用户主动操作时请求道路吸附 |
| 通知 | 显示持续模拟的前台服务通知 |
| Health Connect 写入步数 | 用户主动开启后同步模拟步数，仅 Android 14+ |

## 安全与使用边界

本项目仅用于测试自己有权测试的应用、导航原型和位置相关功能。第三方应用可能主动
识别并拒绝模拟位置，这是 Android 安全模型的一部分。

应用使用 Android 官方测试位置提供器。普通地图、导航原型或自行开发的应用，若使用
标准 Android Location API 且没有禁止模拟位置，通常可以收到模拟坐标。但是，目标
应用仍可通过位置对象的模拟标记、系统 AppOps、设备完整性结果、轨迹一致性或自身风控
策略识别并拒绝模拟位置。因此无法保证“所有软件”都接受模拟坐标。

本项目不会隐藏模拟标记、伪造设备完整性、修改其他应用、注入硬件传感器，或绕过目标
应用的安全检测。若某个自有测试应用拒绝模拟位置，应在该应用的测试构建中显式允许
mock provider，或使用 Android 模拟器和自动化测试环境。

请遵守所在地法律、目标服务条款和设备管理规则。项目按原样提供，维护者不对滥用造成
的后果负责。
