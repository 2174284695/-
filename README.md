# 路线模拟器

用于 Android 开发、导航原型和定位场景测试的开源路线模拟应用。

应用通过 Android 官方的“选择模拟位置信息应用”机制工作，不需要 root，也不包含
隐藏模拟标记、绕过第三方应用检测或规避 Android 安全机制的功能。

## 功能

- 使用 MapLibre 浏览地图，支持手势和按钮缩放至 20 级
- 在地图上点选路线，或用手指连续绘制路线
- 自动抽稀手绘轨迹，减少无意义的密集坐标点
- 按每次点选或每一笔手绘撤销
- 将绘制路线吸附为步行道路或驾车道路
- 一键定位到手机当前真实位置
- 命名保存路线，并在路线库中载入或删除
- 自定义目标速度（1.0～30.0 km/h）
- 自定义自然速度波动（±1%～±15%）
- 使用平滑随机漂移和加速度限制模拟自然移动
- 根据实时移动距离和动态步幅累计模拟步数
- Android 14+ 可在用户授权后将步数同步到 Health Connect
- 使用前台服务持续模拟，可从通知栏停止
- 自动保存当前路线、路线库、速度和波动设置
- 应用回到前台时自动恢复地图，必要时重新加载地图样式

## 系统要求

- Android 8.0（API 26）或更高版本
- 已开启系统开发者选项
- 可访问互联网，以加载在线地图
- Android Studio 自带的 JDK 17 或其他兼容 JDK

Health Connect 步数同步仅支持 Android 14（API 34）或更高版本，并且需要用户明确授权。

## 使用方法

1. 使用 Android Studio 打开项目并安装 `app`。
2. 在手机中开启“开发者选项”。
3. 进入“选择模拟位置信息应用”，选择“路线模拟器”。
4. 开启系统定位服务，并授予应用精确位置权限。
5. 点击地图右侧“定位”，可直接移动到手机当前真实位置。
6. 选择“点选”或“手绘”，在地图上创建路线。
7. 点击“道路吸附”，选择“步行道路”或“驾车道路”，可把折线转换为沿道路路线。
8. 点击“保存路线”可命名保存，并可从“路线库”再次载入。
9. 设置速度和自然波动幅度，然后点击“开始模拟”。

停止模拟后，应用会移除测试位置提供器，让系统恢复真实定位。

如果从 1.0 版本升级后，应用仍未出现在“选择模拟位置信息应用”列表中，请卸载旧版
后重新安装。1.1 及更高版本已经声明 Android 所需的模拟位置权限。

## 模拟步数

应用根据路线的实际推进距离和当前速度估算动态步幅，并在界面中实时累计模拟步数。
在 Android 14 或更高版本上，可以开启“同步模拟步数到 Health Connect”，授权后应用
会以自身为数据来源写入短时间段步数记录。

普通 Android 应用不能向系统硬件 `TYPE_STEP_COUNTER` 传感器注入事件。因此，不支持
Health Connect 或只读取硬件计步传感器的第三方应用，不一定会显示这些模拟步数。

## 地图与数据

项目使用：

- [MapLibre Native Android](https://maplibre.org/maplibre-native/docs/book/android/)
- [OpenFreeMap Liberty](https://openfreemap.org/) 在线矢量地图样式
- [OpenMapTiles](https://openmaptiles.org/) 地图技术
- [OpenStreetMap contributors](https://www.openstreetmap.org/copyright) 地图数据

默认地图样式地址：

```text
https://tiles.openfreemap.org/styles/liberty
```

可以在 `MainActivity.java` 中替换 `MAP_STYLE_URI`，接入自建 MapLibre 样式或其他兼容
服务。

## 道路吸附

绘制至少两个路线点后，点击“道路吸附”并选择出行方式。应用会：

1. 从当前路线中按顺序选取最多 25 个代表点。
2. 请求 OpenStreetMap.de 提供的 OSRM 步行或驾车路由服务。
3. 使用服务返回的完整道路几何替换当前路线。
4. 把吸附前路线保留在撤销历史中，点击“撤销”即可恢复。

道路吸附需要联网。公共路由服务可能限流、暂时不可用，或因当地道路数据不完整而无法
生成理想路线。大量使用或正式部署时，应改为自建 OSRM 服务或接入具有明确服务保障的
路由供应商。默认端点定义在 `RoadRouteClient.java`。

## 数据与隐私

路线、路线库和设置默认保存在设备本地，不会发送给项目维护者。

- 地图服务会收到加载地图切片所需的常规网络请求。
- 只有用户主动点击“道路吸附”时，最多 25 个代表性路线坐标才会发送到
  `routing.openstreetmap.de`，用于计算道路路线。
- 开启 Health Connect 同步后，估算步数会按用户授权写入设备上的 Health Connect。

## 权限说明

| 权限 | 用途 |
| --- | --- |
| 精确位置 | 一键定位真实位置，以及运行 Android 定位前台服务 |
| 模拟位置 | 被 Android 开发者选项选为模拟位置应用 |
| 网络和网络状态 | 加载在线地图，并在用户主动操作时请求道路吸附 |
| 通知 | 显示持续模拟的前台服务通知 |
| Health Connect 写入步数 | 用户主动开启后同步模拟步数，仅 Android 14+ |

## 安全与使用边界

本项目仅用于测试自己有权测试的应用、导航原型和位置相关功能。第三方应用可能主动
识别并拒绝模拟位置，这是 Android 安全模型的一部分。

应用使用 Android 官方测试位置提供器。普通地图、导航原型或自行开发的应用，若使用
标准 Android Location API 且没有禁止模拟位置，通常可以收到模拟坐标。但是，目标
应用仍可通过位置对象的模拟标记、系统 AppOps、设备完整性结果、轨迹一致性或自身风控
策略识别并拒绝模拟位置。因此无法保证“所有软件”都接受模拟坐标。

本项目不会隐藏模拟标记、伪造设备完整性、修改其他应用、注入硬件传感器，或绕过目标
应用的安全检测。若某个自有测试应用拒绝模拟位置，应在该应用的测试构建中显式允许
mock provider，或使用 Android 模拟器和自动化测试环境。

请遵守所在地法律、目标服务条款和设备管理规则。项目按原样提供，维护者不对滥用造成
的后果负责。
