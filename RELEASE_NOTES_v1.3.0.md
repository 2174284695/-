# 路线模拟器 v1.3.0

本版本新增道路吸附，并补充模拟位置兼容性、数据发送范围和公共路由服务限制说明。

## 新增功能

- 新增“道路吸附”按钮
- 支持生成步行道路路线
- 支持生成驾车道路路线
- 从长路线中按顺序选择最多 25 个代表点，避免请求地址过长
- 使用路由服务返回的完整 GeoJSON 道路几何
- 吸附前路线自动进入撤销历史，可一键恢复
- 吸附期间锁定路线编辑，避免异步结果覆盖用户的新修改
- 请求失败时保留原路线，并显示具体错误

## 使用方法

1. 通过“点选”或“手绘”绘制至少两个路线点。
2. 点击“道路吸附”。
3. 选择“步行道路”或“驾车道路”。
4. 等待路线替换完成，再设置速度并开始模拟。

如果吸附结果不合适，可以点击“撤销”恢复原路线。

## 道路服务与隐私

道路吸附使用 OpenStreetMap.de 提供的 OSRM 公共路由服务。只有用户主动点击吸附时，
应用才会发送最多 25 个代表性路线坐标。路线库和未执行吸附的绘制路线仍保存在设备
本地，不会发送给项目维护者。

公共服务可能限流、暂时不可用，且不适合作为高频或商业生产服务。正式部署建议自建
OSRM 服务或替换为具有服务保障的路由供应商。

## 模拟位置兼容性

应用通过 Android 官方测试位置提供器工作。使用标准 Location API 且未禁止模拟位置
的应用通常可以收到模拟坐标，但不能保证所有第三方软件都接受。

第三方应用可以检查模拟位置标记、AppOps、设备完整性、传感器和轨迹一致性，或使用
自身风控策略拒绝数据。本项目不会隐藏模拟标记、伪造完整性结果、修改第三方应用或
绕过其安全检测。

## 已知限制

- 道路吸附依赖网络连接以及公共路由服务的可用性。
- 道路结果取决于 OpenStreetMap 数据质量，部分区域可能不完整。
- 代表点最多为 25 个，极长或非常复杂的手绘路线可能与原始意图存在偏差。
- 普通应用不能向硬件 `TYPE_STEP_COUNTER` 注入事件。
- 默认在线地图和道路服务在部分网络环境下可能无法访问。

## 安装

1. 安装发布页中的 APK。
2. 在 Android 开发者选项中选择“路线模拟器”为模拟位置应用。
3. 开启系统定位服务并授予精确位置权限。

系统要求：Android 8.0（API 26）或更高版本。
